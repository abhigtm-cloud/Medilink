## LinkedIn Post Caption - Option 3 (Technical Achievement)

---

**Just shipped MediLink v1** 🚀

A real-world case study in building reliable mobile apps for healthcare.

**Architecture highlights:**

🔄 **Intelligent Caching:** Hive-based local storage with 10-minute TTL reduces Firebase calls by 90%

⏱️ **Resilient Authentication:** 60s timeout + exponential backoff retry logic (3 attempts) handles slow regional networks

🗺️ **Real-time Location:** Integrated Geolocator + Geocoding for reverse address lookup and emergency hospital discovery

 **Scalable DB:** Firebase Realtime Database with query optimization & batch operations

**Performance Metrics:**
- Hospital list load: 500ms (cached), 2-3s (first load)
- Booking confirmation: <1s
- Location geocoding: 1-2s
- APK size: 56MB (optimized with tree-shaking)

**Challenges Solved:**
✅ Regional latency (implemented caching layer)
✅ Slow 3G networks (extended timeouts + retry logic)
✅ Real-time slot management (Firebase transactions)
✅ Emergency response (location-based hospital discovery)

**Stack:** Flutter • Firebase • Riverpod • Hive • Google Maps

Now focusing on UAT and scaling user base.
